3. Create synthetic demo images
------------------------------------------------------------

This avoids using private clinical data.

python - <<'PY'
from pathlib import Path
from PIL import Image, ImageDraw

input_dir = Path("data/demo_cli_input")
input_dir.mkdir(parents=True, exist_ok=True)

for i in range(1, 4):
    img = Image.new("RGB", (600, 400), "white")
    draw = ImageDraw.Draw(img)
    draw.rectangle((40, 40, 560, 360), outline="gray", width=3)
    draw.text((60, 70), f"SYNTHETIC DEMO IMAGE {i}", fill="black")
    draw.text((60, 110), "VISIBLE ID: DEMO_PATIENT_123", fill="black")
    draw.text((60, 150), "This is not real patient data.", fill="black")
    img.save(input_dir / f"demo_image_{i:02d}.png")
print(f"Created synthetic images in: {input_dir.resolve()}")
PY

Check:

dir data\demo_cli_input

Expected:

demo_image_02.png
demo_image_03.png

10. Demo talking points
------------------------------------------------------------

Say:
1. The CLI is useful for technical users and batch processing.
2. It can run locally without uploading sensitive images.
3. Rectangles can censor burned-in identifiers.
4. Metadata anonymization and pixel censoring are separate privacy layers.
5. Synthetic data is used in this demo to avoid exposing private clinical data.

------------------------------------------------------------
11. Troubleshooting
------------------------------------------------------------

If the CLI executable is missing:
python -m PyInstaller --clean --noconfirm clinical_image_anonymizer_cli.spec

If the command options differ:
.\dist\clinical-image-anonymizer.exe --help

If output is not created:
Check the input path, output path, and rectangle syntax.

If Windows blocks the exe:
Right-click the exe > Properties > Unblock, if shown.
